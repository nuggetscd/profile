我用过的 AI 工具
以及我怎么看它们

赵想哲 NLP 硕士在读 @ UCLouvain 2026.03
我不只是这些工具的用户——我会把它们组合起来解决实际问题。下面是我用下来真实的感受，不是参数对比，而是用的时候到底是什么体验。
Claude LLM Coding

我目前主力使用的模型。写代码的能力很强，多任务处理也很稳——能在一次对话里同时改代码、写文档、分析问题，不容易跑偏。在 Claude Code 里的体验和 Cursor 差不多，是真正能用来干活的 AI coding 工具。

让我印象最深的是它的意图理解。问同一个问题，ChatGPT 和 Gemini 会先总结你说的话，然后分点展开，最后再收尾——格式看起来很"专业"，但实际上是在用结构掩盖对意图的判断。Claude 不一样：如果你要的是一句话答案，它就给一句话；如果你要的是完整分析，它才展开。在我复习课程内容时，有问题直接给出精确解答，不绕弯子。

这个差异背后是产品逻辑的不同：ChatGPT/Gemini 的输出格式服务于"看起来有条理"，Claude 的输出格式服务于用户真正想要的东西。前者迎合预期，后者理解意图。

我的判断 意图驱动输出，而不是模板驱动输出。这是它和其他模型最本质的区别。
Gemini LLM 生态

会员可以无限用 token，这是它最大的优势。任务类型也很多样。但它有个让我困扰的习惯——简单的问题说很长，经常绕一大圈才到重点。我判断这主要是 prompt 风格的问题，调整一下会好很多，但默认体验不够干净。

真正让我觉得 Gemini 不可替代的是 Google 生态的整合：Google Drive、语音转录、YouTube 摘要、搜索……这些工具本来就在同一个生态里，做项目时不需要来回切换，效率很高。如果你重度依赖 Google 全家桶，Gemini 值得认真用。

我的判断 靠生态取胜，模型本身中规中矩。token 无限是刚需场景的救星。
豆包 LLM

我用得相对少，整体能力和 Claude、Gemini 这些主流产品还有差距。但它在中国互联网的传播力很强——抖音上各种豆包对话截图、角色扮演视频流量很高，以搞笑和娱乐内容为主，出圈靠的是社交属性而不是技术深度。

这个观察本身挺有意思：产品能力和传播力可以完全脱钩。豆包是个很好的案例，说明在中国市场，内容玩法和平台分发比模型本身更决定用户规模。

我的判断 适合娱乐向场景。研究中国 AI 产品的市场策略，豆包是个好样本。
Cursor Coding

我把 Cursor 理解成一个有极好交互层的 AI 接入器——它本质上是用 API token 连接不同模型来干活，模型本身可以随时换。它的价值不在于模型，在于交互设计。

最让我印象深刻的是做游戏迭代的时候：修改代码之后可以直接在 Cursor 里看到游戏运行效果，边看边改，反馈循环极短。这种"看见 → 改 → 立刻再看见"的体验，是纯文字对话做不到的。如果说 Claude 是我的思考伙伴，Cursor 就是我的施工现场。

我的判断 交互是核心竞争力。写代码、做项目迭代，体验目前最好。
即梦 图像生成

用来生成项目所需的图像素材。效果还可以，但严重依赖 prompt 质量——描述得模糊，出来的东西就很随机；描述得精确，才能稳定出想要的结果。用它的过程让我对"文生图 prompt 工程"有了具体的认识：不只是说"画一张什么"，而是要控制风格、构图、光线、细节层次。

不过即梦有个我觉得做得聪明的地方：同一个 prompt 每次生成四张，结果都不一样，不满意可以直接重新生成。这本质上是在用产品设计消化模型的随机性——文生图模型天然不稳定，"每次结果不同"是技术局限，但包装成"提供多种可能性"就成了功能。把约束转化成特性，这个决策值得学习。

我的判断 用产品设计消化了模型的不确定性。会用 = 会写 prompt，更会理解生成结果。
